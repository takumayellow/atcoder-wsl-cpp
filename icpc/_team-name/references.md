## 「Rinshan Solution」が特別な理由の分析

嶺上開花という概念が持つ意味：
- 槓（カン）という通常とは異なる手順を踏んで
- 余分に引いた牌でアガる
- = **予期しないルートから解を得る**

「Solution」を「開花」に当てはめると：
- 「嶺上」（予期しないルート）が残る
- = 「予期しないアプローチからの解」という意味が成立する

**機能する条件**：
1. 元の概念が「一つの状況・感情・体験」を表している
2. その概念の一部をCS用語に置き換えても意味が通る
3. 置き換え後に**元の意味が新しいコンテキストで生きる**

## 機能しない名前のパターン

| 名前 | 問題点 |
|------|--------|
| Bocchi the Stack | 「Bocchi」と「Stack」に意味的な接点がない。音の置き換えに過ぎない |
| K-Nearest Neighbor | 発音が似てるだけ。K-Onの意味とK-NNの意味が接続しない |
| Homura Break | breakという動詞は合うが、「暁ほむら」の本質（ループ・記憶・犠牲）が出ない |
| Dango Array | 見た目は似てるが、配列とだんごの共通点はそれだけ |

**共通の失敗パターン**: 「キャラ名 + CSワード」の単純結合

## 良い名前の構造パターン

### パターンA: 意味が転写される（Rinshan型）
- 元の概念の「感情・体験・状況」がCS世界でも正確に機能する
- 例: 嶺上開花 → 予期しないルートからの解

### パターンB: 音のギミック（Yare Yare Deque型）  
- 音が似ている＋**意味も偶然合っている**（両方成立することが重要）
- 「やれやれ」の疲れ感とdequeを扱う煩わしさが一致

### パターンC: 逆説・アイロニー（Trivially AC型）
- 数学の「自明」「証明は読者の演習として残す」等の慣用句をCP文脈に転用
- 知っている人間には即座に刺さる

### パターンD: 圧縮された物語（tonosama型）
- 一言で「俺たちのスタンス」を表す
- 説明しなくてもキャラが立つ

## 参考: 良いひねりの「種」になりそうな概念

### 麻雀（咲）
- 海底摸月（ハイテイ）: 最後の一牌でアガる = ギリギリAC
- 一発: リーチ後最初のツモでアガる = 一発AC
- 振り込み: 相手の当たり牌を捨てる = 自分でバグを踏む
- 聴牌（テンパイ）: あと一枚でアガれる = あと一歩でAC
- 流し満貫: 捨て牌で勝つ = 捨てたアプローチで勝つ

### 数学の慣用句
- "Trivially" : 自明（実際は全然自明でない）
- "Left as an exercise" : 証明は読者への課題
- "Without loss of generality" (WLOG) : 一般性を失わずに
- "QED" : 証明終了
- "Proof by contradiction" : 背理法
- "Sufficiently large N" : 十分大きいNに対して
- "Assume P = NP" : P=NPを仮定する

### CP特有の感情
- WA→WA→WA→AC の流れ
- 最後の1秒でAC
- 全探索が通る（通るはずがないのに）
- 解法は合ってるがバグがある状態
- editorialを見たら自分の解法が合ってた
