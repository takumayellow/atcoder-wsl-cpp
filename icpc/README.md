# ICPC 用作業フォルダ

チーム（C++ 2〜3人 ＋ Python 1人 / OS バラバラ）の共有環境。
AtCoder 用とは別物。**C++ と Python を同等**に整備する。

公式: https://icpc.jp/ ／ 2026 国内予選ルール: https://icpc.jp/2026/domestic/rules/

---

## 1. 2026 国内予選ルール要点（要・全員確認）

| 項目 | 内容 |
|------|------|
| **チーム** | 3人で **1台のPC**（マウス/キーボード/ディスプレイ/プリンタ 各1台のみ） |
| **言語** | **C / C++ / Java / Kotlin / Python 3** のみ |
| **インターネット** | **競技システム以外アクセス禁止**（例外: システムURL確認のためのメール参照のみ） |
| **持ち込み** | ✅ **印刷物**（手書きメモ・参考書・辞書）OK ／ ✅ **事前にPCへ置いたファイル/ソフト**OK |
| **外部ライブラリ** | **ソースに同梱して提出コード単体でコンパイル可能なら OK** |
| **生成AI** | ❌ Copilot 等のコード自動生成ツールは**禁止** |
| **開催** | 2026-07-03(金) 16:30–19:30 予定（各大学キャンパスからオンライン）※日時は公式で要再確認 |

### ライブラリの実務的な結論

- **AC Library (C++)** … ✅ ヘッダオンリーなので、使う部分を**ソースに貼れば使える**（→ `library/cpp/` に抜粋を常備）
- **Boost (C++)** … △ 同梱は理屈上可だが巨大。必要部分だけ抜き出す運用
- **NumPy (Python)** … ❌ **実質使えない**（C拡張で同梱コンパイル不可・ジャッジ非搭載前提）→ **素の Python 3** で書く

> ⚠ ネット不可なので、当日は **印刷したチームノート** か **事前にローカル配置したスニペット** を
> **コピペ**して使う。GitHub 等から当日取ってくることはできない。

## 2. フォルダ構成

```
icpc/
├── README.md            # このファイル（ルール要点・使い方）
├── template/            # 複数データセット入力の雛形
│   ├── main.cpp         #   C++（高速IO・番兵/EOF/ケース数の3パターン）
│   └── main.py          #   Python（素の Python 3、NumPy不可前提）
├── library/             # チームノートの種（標準のみ＋同梱可なもの）
│   ├── README.md        #   収録物の目次
│   ├── cpp/             #   ACL抜粋（dsu, fenwick, segtree…）＋自前
│   └── py/              #   DSU, Fenwick, SegTree… を素 Python で
├── notebook/            # library/ を印刷用 PDF にビルド（本番持ち込み用）
├── 練習会/              # チーム練習・模擬（セッションごとにサブフォルダ）
└── 本番/                # 公式戦
    └── 2026-国内予選/   #   当日の作業フォルダ
```

## 3. 使い方

### 問題を解くとき
1. `練習会/` か `本番/2026-国内予選/` の中に問題ごとのフォルダ（`A/`, `B/`, …）を作る
2. `template/main.cpp` または `template/main.py` をコピーして開始
3. ライブラリが要るときは `library/` から該当スニペットを**貼り付ける**

### 本番前
- `notebook/` で `library/` を **印刷用 PDF** にして紙で持ち込む（ネット不可対策）
- 事前に PC にも `library/` を置いておく（ルール上 OK）

### 既存ツールの流用
- ローカルテストは AtCoder と同じ `oj`（`actest` 等）が使える（ジャッジは別だがローカル検証は同じ）
- 過去問演習は **AOJ**（会津大学オンラインジャッジ）。AtCoder とは別ジャッジな点に注意

## 4. 提出形式と入力（重要）

公式2026ルール原文: 「競技者はプログラムの**ソースコード**をコンテストシステムに提出します」
（compiled and executed on the contest server）。

- **入力**: 1問 = 1ファイルに**複数データセットが連結**。末尾は **`0`（や `0 0`）の行で終端**。
- **出力**: 「各データセットについて出力せよ」= **1本のプログラムで全データセットをループ**し、
  各反復で1つずつ出力する（= `template/` のパターンA）。**データセットごとに別コードを出す必要はない**。
- **検証**: その1ファイルを丸ごと食わせて**一気に**全データセットを処理して確認できる。

> ⚠ **チームで要確認**: 2024年以降の参加記には「データセットを**DLして手元実行**し、**6分以内に
> 出力を提出**」という別系統の記述があり、公式の「ソースコード提出・サーバ実行」と食い違う。
> **書くコードはどちらでも同じ（データセットをループする1本）**だが、**当日の提出機構（コード提出か
> 出力提出か・6分制限の有無）は公式マニュアルPDFで最終確認**すること。

## 5. 本番で気をつけること

### ルール違反になる（必須）
- **生成AI補完を完全オフ**: Copilot / Codeium / Tabnine / Continue / Cline 等。ルールで
  「生成AIシステム（GitHub Copilot等）」は**禁止**。
- **VSCode の Settings Sync でサインインしない**: 共有/貸出PCで自分のアカウントに入ると
  Copilot や拡張が**同期で勝手に入る**＝事故。クリーンな既定プロファイルで臨む。
- **インターネット遮断**: 競技システム以外アクセス禁止（ブラウザは閉じる）。

### 推奨（事故防止）
- **色テーマ・設定はデフォルトのまま**、個人設定を持ち込まない。`format on save` も切る
  （出力とコードの取り違え・無駄な差分防止）。`.vscode/settings.json` で AI 補完オフのみ最小設定。
- **非AIの IntelliSense（標準補完）は使ってOK**（IDE使用は公式に許可。切るのは AI のみ）。
- **出力の完全一致**: 行末スペース・改行・大文字小文字までファイル全体で照合 → 1文字ずれで WA。
- **終端 `0` を出力しない／正しく読み飛ばす**。
- **コンパイラ/言語バージョンを事前に本番ジャッジへ合わせる**（手元 g++・Python のバージョン確認）。
- **3人で1台** → テンプレ・キーバインド・エディタを事前統一。
- **印刷チームノート必携**（ネット不可）。`library/` を紙＋ローカル両方に。
